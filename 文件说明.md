# dog_emo_repro 项目文件说明

这份文档用于整理 `D:\newdog_emo\dog_emo_repro` 里现有文件的用途。只做说明，不移动、不删除任何文件。

当前实验主线是：

```text
STFT + HL-FE + BiGRU fallback
```

当前主线训练入口是：

```text
src/train.py
```

推荐优先阅读和维护 `src/` 目录里的代码。根目录下的 `train_wavlm_dog.py`、`model.py`、`dataset.py` 等主要是早期 WavLM 分支或旧实验代码。

## 当前主线

### src/train.py

当前最重要的训练脚本。

负责：

- 扫描或读取数据索引
- 划分 train/test split
- 构建 STFT / SST-STFT 数据集
- 构建模型
- 训练 STFT + HL-FE + BiGRU fallback
- 支持 `ce`、`weighted_ce`、`focal` 三种 loss
- 保存 `best_model.pt` 和 `last_model.pt`
- 使用 `best_model.pt` 做最终评估
- 保存训练日志、混淆矩阵、分类报告、指标 JSON、曲线图

当前推荐命令示例，PowerShell 版本：

```powershell
cd D:\newdog_emo\dog_emo_repro

D:\py310\python.exe src\train.py `
  --data_dir "D:\wlmdog" `
  --project_dir "D:\newdog_emo\dog_emo_repro" `
  --feature_type stft `
  --use_hl_fe `
  --backend bigru_fallback `
  --epochs 20 `
  --loss_type focal `
  --focal_gamma 1.5 `
  --lambda_hl_prior 0 `
  --weight_decay 1e-3 `
  --dropout 0.5 `
  --patience 4
```

CMD / bat 版本续行符是 `^`，PowerShell 续行符是反引号 `` ` ``。

### src/dataset.py

当前主线的数据集读取代码。

负责：

- 从 split CSV 读取音频路径和标签
- 优先用 `soundfile` 读取音频
- `soundfile` 失败后 fallback 到 `torchaudio`
- 双声道转单声道
- 检查 sample rate
- 必要时重采样到目标采样率
- padding / truncate 到固定长度
- 调用 STFT / SST-STFT 特征提取
- 可选读取和保存 feature cache

这是当前音频读取 warning 修复的位置。

### src/features.py

当前主线的特征提取代码。

负责：

- STFT 特征提取
- SST-STFT 特征提取
- STFT fallback 相关逻辑
- 特征缓存路径生成
- 统一音频参数，例如采样率、时长、FFT 参数

当前主线使用：

```text
--feature_type stft
```

### src/models.py

当前主线的模型定义。

包含：

- `SimpleCNNBaseline`：CNN baseline，当前主线不使用
- `HLFE`：高低频特征融合模块
- `_FallbackSequenceEncoder`：BiGRU fallback 编码器
- `MambaEmotionModel`：历史命名仍叫 MambaEmotionModel，但在 `--backend bigru_fallback` 下实际使用 BiGRU fallback

当前 HL-FE 设置：

```text
hl_logits = [0.3, 0.0]
temperature = 0.7
初始 low/high 约为 0.6055 / 0.3945
```

注意：当前主线不要改成 CNN / Transformer / WavLM / 真 Mamba。

### src/evaluate.py

旧的评估入口。

负责：

- 从 `outputs/checkpoints/best_model.pt` 这类旧路径加载模型
- 在测试集上评估
- 生成报告和可视化

当前针对单个 run 的评估，优先使用根目录的 `evaluate_stft_hlfe.py`。

### src/visualize.py

旧的可视化脚本。

负责：

- 数据集分布图
- 时长分布图
- 样本波形 / 频谱图
- 混淆矩阵等辅助图

不是当前训练主流程必须入口，但 `run_eval.bat` 会调用它。

### src/scan_dataset.py

数据集扫描脚本。

负责：

- 扫描数据目录下的类别文件夹
- 记录 `.wav` 文件路径、类别、时长、采样率
- 生成 `outputs/reports/dataset_index.csv`
- 记录坏文件到 `bad_files.csv`

### src/utils.py

当前 `src/` 主线使用的工具函数。

负责：

- 创建项目输出目录
- 设置随机种子
- 选择设备 CPU / CUDA
- 波形归一化
- 从 DataFrame 提取类别名

## bat 脚本

这些 `.bat` 多数是历史快捷命令。有些仍可参考，但当前推荐直接用 `src/train.py` 的显式命令。

### run_train.bat

旧的长训练入口。

运行：

```text
src/train.py --model mamba --use_hlfe --epochs 100
```

不建议作为当前默认实验命令，因为参数没有显式固定 focal / dropout / weight_decay / patience。

### run_train_fast.bat

快速小实验入口。

运行 5 epoch、小 batch，用于快速检查流程是否能跑通。

名字里有 `mamba`，但如果没有真 Mamba 或指定 fallback，实际可能走 BiGRU fallback。当前更推荐在命令行显式写：

```text
--backend bigru_fallback
```

### run_train_stft_mamba_nohlfe_20ep_cpu.bat

STFT + 无 HL-FE 对照实验脚本。

当前主线要求使用 HL-FE，因此只作为历史对照。

### run_train_stft_cnn_30ep_cpu.bat

STFT + CNN baseline 脚本。

当前主线不使用 CNN。

### run_train_sst_mamba_hlfe_20ep_cpu.bat

SST-STFT + HL-FE 训练脚本。

当前主线是 STFT，不是 SST-STFT。

### run_eval.bat

旧评估和可视化快捷脚本。

会运行：

- `src/evaluate.py`
- `src/visualize.py`

当前针对某个 run 的精确评估，优先使用 `evaluate_stft_hlfe.py`。

### run_eval_sst_mamba_hlfe.bat

## 当前实验怎么比较

比较 `weighted_ce` 和 `focal` 时，优先比较两个 run 目录中的：

```text
metrics.json
classification_report.txt
confusion_matrix.csv
loss_curve.png
macro_f1_curve.png
hl_weights_log.csv
```

重点指标：

- `test_acc`
- `macro_f1`
- happy recall
- sad recall
- happy 被误判成 angry / anxious 的数量
- sad 被误判成 anxious / lonely 的数量
- `val_loss` 是否稳定
- HL-FE 的 `low_weight` / `high_weight` 是否真的发生变化
- `hl_logits_grad_norm` 是否长期接近 0
- `hl_logits_delta_norm` 是否太小

如果 accuracy 差不多，但 macro F1、happy recall、sad recall 提升，这次 focal loss 仍然有价值。

## 命名提醒

项目里一些历史名字容易误导：

- `MambaEmotionModel` 是历史类名；在 `--backend bigru_fallback` 时实际是 BiGRU fallback。
- `run_train_stft_mamba_*.bat` 是历史文件名；当前主线应显式指定 `--backend bigru_fallback`。
- 根目录 `dataset.py` 和 `src/dataset.py` 不是同一个文件；当前主线使用 `src/dataset.py`。
- 根目录 `model.py` 是 WavLM 模型；当前主线模型在 `src/models.py`。
- `checkpoints/best_model.pt` 是早期 WavLM 分支；当前主线 checkpoint 在每个 `runs/stft_hlfe_bigru_*/` 目录下。
